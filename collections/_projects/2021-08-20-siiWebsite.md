**Web Architecture · Maintainability · Static Sites**

Refactored and stabilised a legacy website by migrating it from a fragmented WordPress/HTML codebase to a maintainable static site architecture.

- Identified issues with a **degraded WordPress-to-HTML migration**, where tightly coupled JavaScript and plugin remnants made updates difficult
- Re-architected the site using **Jekyll**, enabling content to be managed through simple Markdown-based workflows
- Removed unnecessary dependencies and **decoupled legacy components**, improving maintainability and clarity of the codebase
- Preserved the existing **design and user experience** while significantly improving internal usability for content updates

- Transformed a brittle system into a **sustainable, team-friendly content pipeline**
- Reduced friction for non-developers to update and maintain the site

_Focused on improving long-term maintainability by simplifying architecture and reducing unnecessary complexity._
